# TODO Tracker

> **Purpose**: Automated TODO/FIXME/HACK comment management across the codebase - scans, categorizes, creates GitHub issues, and tracks completion over time.

**Version**: 1.0.0
**Type**: Workflow
**Invocation**: `/todo-tracker [options]`

---

## Overview

The TODO Tracker is a code maintenance tool that automatically discovers, categorizes, and manages TODO/FIXME/HACK comments across the entire codebase. It helps prevent technical debt accumulation by surfacing critical TODOs, creating GitHub issues for high-priority items, and tracking completion progress over time.

**CRITICAL**: Based on Phase 8 audit findings, untracked TODOs scatter across the codebase create hidden technical debt. This skill provides automated tracking, ensuring critical TODOs don't get lost during rapid development cycles.

## Usage

### Basic Usage

```bash
# Scan entire codebase for TODOs
/todo-tracker

# Scan specific module
/todo-tracker --module apps/admin

# Filter by severity
/todo-tracker --severity critical

# Create GitHub issues for high-priority TODOs
/todo-tracker --action create-issues --severity high

# Generate markdown report
/todo-tracker --action report --format markdown
```

### CI/CD Integration

```yaml
# .github/workflows/todo-tracking.yml
name: TODO Tracking

on:
  schedule:
    - cron: '0 0 * * 0'  # Weekly on Sunday
  push:
    branches: [main]

jobs:
  track-todos:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
      - run: pnpm install
      - name: Scan TODOs
        run: /todo-tracker --action scan --format json > todos.json
      - name: Create Issues for Critical TODOs
        run: /todo-tracker --action create-issues --severity critical
      - name: Upload Report
        uses: actions/upload-artifact@v3
        with:
          name: todo-report
          path: todos.json
```

### Pre-Commit Hook

```bash
# .husky/pre-commit
#!/bin/sh
. "$(dirname "$0")/_/husky.sh"

# Check for new critical TODOs in staged files
staged_files=$(git diff --cached --name-only --diff-filter=ACM | grep '\.tsx\?$\|\.jsx\?$')

if [ -n "$staged_files" ]; then
  echo "🔍 Checking for critical TODOs..."

  for file in $staged_files; do
    critical=$(grep -in "TODO.*CRITICAL\|FIXME.*CRITICAL" "$file" || true)

    if [ -n "$critical" ]; then
      echo "❌ Critical TODO found in $file:"
      echo "$critical"
      echo ""
      echo "Create an issue or resolve before committing."
      exit 1
    fi
  done

  echo "✅ No critical TODOs in staged files"
fi
```

## Workflow Actions

### Action 1: Scan TODOs

**Command**: `/todo-tracker --action scan`

**What it does**:
1. Scans all TypeScript/JavaScript/Markdown files
2. Finds TODO, FIXME, and HACK comments
3. Categorizes by severity (critical, high, medium, low)
4. Categorizes by module (api, ui, database, etc.)
5. Saves tracking data to `.claude/todo-tracking/`

**Output**:
```
🔍 Scanning for TODOs...

📊 TODO Summary:

  Total TODOs: 247
  Critical: 12
  High: 45
  Medium: 134
  Low: 56

🚨 CRITICAL (12)

  📄 apps/admin/app/api/orders/route.ts:42
     // TODO CRITICAL: Fix SQL injection vulnerability in order search
     Category: api

  📄 packages/db/src/client.ts:123
     // FIXME CRITICAL: Connection pool exhaustion under load
     Category: database

💾 Tracking data saved: .claude/todo-tracking/2026-02-27.json
```

---

### Action 2: Create GitHub Issues

**Command**: `/todo-tracker --action create-issues --severity high`

**What it does**:
1. Runs scan to find TODOs
2. Filters for critical and high severity
3. Creates GitHub issues using `gh` CLI
4. Adds labels: `tech-debt`, `priority-high`/`priority-medium`, category
5. Links to file location

**Output**:
```
🎫 Creating GitHub Issues for TODOs...

📝 Creating 12 GitHub issues...

✅ Created: https://github.com/org/repo/issues/456
   [TODO] Fix SQL injection vulnerability in order search

✅ Created: https://github.com/org/repo/issues/457
   [FIXME] Connection pool exhaustion under load

📊 Summary:
  Created: 12
  Failed: 0
```

**Generated Issue Format**:
```
**Location**: `apps/admin/app/api/orders/route.ts:42`
**Severity**: critical
**Category**: api

```
// TODO CRITICAL: Fix SQL injection vulnerability in order search
```

---
Auto-generated by TODO Tracker
```

## Severity Categorization

TODOs are automatically categorized by severity based on keywords:

| Severity | Keywords | Usage |
|----------|----------|-------|
| **Critical** | `CRITICAL`, `URGENT`, `MUST FIX`, `SECURITY` | Security issues, data loss risks, production blockers |
| **High** | `FIXME`, `HACK`, `IMPORTANT` | Bugs, workarounds, important improvements |
| **Medium** | `SHOULD` | Nice-to-have improvements, refactoring |
| **Low** | (default) | Minor improvements, optimizations |

**Examples**:
```typescript
// TODO CRITICAL: Fix SQL injection vulnerability
// → Severity: critical

// FIXME: Connection pool leaks memory
// → Severity: high

// TODO: Should refactor this to use async/await
// → Severity: medium

// TODO: Add dark mode support
// → Severity: low
```

## Module Categorization

TODOs are categorized by codebase location:

| Category | Path Pattern |
|----------|-------------|
| **api** | `/api/` directories |
| **ui** | `/components/` directories |
| **database** | `packages/db` |
| **authentication** | `packages/auth` |
| **testing** | `.test.`, `.spec.` files |
| **migration** | `/migrations/` directories |
| **other** | Everything else |

## Options

| Option | Description | Default |
|--------|-------------|---------|
| `--action <type>` | Action to perform: scan, create-issues, report, clean | `scan` |
| `--severity <level>` | Filter by severity: all, critical, high, medium, low | `all` |
| `--module <path>` | Filter by module (e.g., apps/admin, packages/db) | All modules |
| `--format <type>` | Output format: text, json, markdown | `text` |
| `--create-issues` | Create GitHub issues for TODOs | `false` |

### Weekly TODO Review (GitHub Actions)

```yaml
name: Weekly TODO Review

  schedule:
    - cron: '0 9 * * 1'  # Monday 9am

jobs:
  todo-review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

        env:

        uses: actions/github-script@v6
        with:
          script: |
            const fs = require('fs')
            const report = fs.readFileSync('.claude/reports/todo-report.md', 'utf8')

            await github.rest.issues.createComment({
              issue_number: context.issue.number,
              owner: context.repo.owner,
              repo: context.repo.repo,
              body: `## Weekly TODO Report\n\n${report}`
            })
```

## Troubleshooting

### Issue: "gh: command not found" when creating issues

**Cause**: GitHub CLI not installed

**Fix**:
```bash
# Install GitHub CLI
brew install gh

# Authenticate
gh auth login
```

---

### Issue: Too many low-priority TODOs cluttering output

**Cause**: Default scan includes all severities

**Fix**: Filter by severity
```bash
# Only show critical and high
/todo-tracker --severity critical
/todo-tracker --severity high

# Or create issues only for important ones
/todo-tracker --action create-issues --severity high
```

---

### Issue: TODOs in test files creating noise

**Cause**: Test files are scanned by default

**Fix**: Skill already excludes `.test.` and `.spec.` files. Check file naming conventions:
```typescript
// Excluded (correct naming)
src/utils.test.ts
src/utils.spec.ts

// Included (incorrect naming)
src/test-utils.ts  // ❌ Not excluded
```

---

### Issue: Historical tracking data growing too large

**Cause**: Daily tracking files accumulate in `.claude/todo-tracking/`

**Fix**: Clean old tracking data
```bash
# Keep last 30 days
find .claude/todo-tracking -name "*.json" -mtime +30 -delete

# Or add to .gitignore
echo ".claude/todo-tracking/*.json" >> .gitignore
```

## Best Practices

### When to Use TODO Comments

**GOOD** (temporary, trackable):
```typescript
// TODO CRITICAL: Fix SQL injection vulnerability (Issue #456)
// TODO: Optimize query performance (Phase 9)
// FIXME: Memory leak in connection pool (investigating)
```

**BAD** (vague, untracked):
```typescript
// TODO: fix this later
// FIXME: doesn't work
// HACK: temporary workaround
```

### Severity Guidelines

**Use CRITICAL for**:
- Security vulnerabilities
- Data loss risks
- Production-breaking bugs
- Performance issues affecting users

**Use HIGH for**:
- Bugs affecting functionality
- Memory leaks, resource exhaustion
- Important refactoring needed

**Use MEDIUM for**:
- Code quality improvements
- Non-critical optimizations
- Nice-to-have features

**Use LOW for**:
- Minor optimizations
- Code style improvements
- Future enhancements
